<article>
	<section class="page">
<p>There are times when you need to associate <em>metadata</em>, or data that describes data, with classes, methods, or other elements in your Java code. For example, your team might need to identify unfinished classes in a large application. For each unfinished class, the metadata would include the name of the developer responsible for finishing the class and the class’s expected completion date.</p><p>Before Java 5, comments were Java's only flexible mechanism for associating metadata with application elements. But because the compiler ignores them, comments are not available at runtime. And even if they were available, you would have to parse the text to obtain crucial data items. Without standardizing how the data items were specified, they might be impossible to parse.</p><p class="jumpTag"><a href="/article/3543240/how-to-describe-java-code-with-annotations.html#jump">To read this article in full, please click here</a></p></section></article>
